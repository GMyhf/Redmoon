# 更新记录

本文件记录 CRIMSON RELAY 每轮迭代的玩法与架构改进。协议层面的字段变化见 [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)。

## 2026-07-14 · 副本 T-003：接入 child worker

- `GameServer` 在副本进入时启动并 attach 受监督的 child worker，按固定主循环 tick 路由输入，接收权威 snapshot/events，并在退出、断线、超时和停服时回收 worker。
- 主 World 继续负责玩家席位、断线保留和 reward-once 结算；worker 副本实体不进入主集合，不发普通 XP/金币/掉落，也不进入主世界 respawn 队列。
- worker 的 `stateVersion` 现在回写副本实例并参与 settle 校验；副本事件按原地图/成员作用域回投。未修改浏览器/Godot 线上协议或 `PROTOCOL_VERSION`。
- 修正 worker 事件归档使用 `event.event` 字段，并补充 worker 完成到服务端 settle、一次性奖励的集成回归。

## 2026-07-14 · 副本 Phase 5：结算幂等与回收

- 主 World 新增带 `settlementId` 和 `stateVersion` 校验的副本结算入口，先登记终态再发放 XP、金币和复苏露；重复请求只返回已登记结果，不会重复奖励。
- child worker/transport 新增 `settle` 请求通道；worker 只提交实例、成员、奖励计划和状态版本，主进程仍是奖励事实来源。
- 超时与 worker 丢失统一走一次性失败回收，成员回城、实体清理和 `dungeonFailed` 事件不会重复执行；未改动线上客户端协议。

## 2026-07-14 · 副本 Phase 0：可恢复随机源

- `World` 默认使用带随机 seed、可序列化状态的 Mulberry32 PRNG，新增 `getRandomState()` /
  `restoreRandomState()`，为副本跨进程检查点恢复保留精确的随机序列。
- 保留现有 `rng: () => number` 测试注入契约；函数注入源明确不可序列化，不会静默回退到 `Math.random`。
- 新增同 seed、保存/恢复状态和 `World` 注入兼容性测试；未改动线上协议。

## 2026-07-14 · 副本 Phase 1：独立进程传输与握手

- 新增带 4 字节长度前缀和最大帧限制的 child process IPC；worker 的 stdout 只承载协议帧，stderr 独立收集。
- 新增 `open`/`ready`、`heartbeat`、`error` 和 `recycle` 生命周期，主进程监督握手超时、异常退出、协议版本和响应身份。
- 新增真实子进程测试，覆盖拆帧、超大帧、正常回收、unsupported message、静默超时、损坏帧和 stderr 隔离；未接入副本业务或线上协议。

## 2026-07-14 · 副本 Phase 2：版本票据与席位校验

- 副本开启时由主进程签发带 schema/protocol、实例 ID、平均等级、队伍成员、逻辑时间、过期时间和序号的 HMAC 票据。
- 票据使用固定字段 canonical JSON 与 `timingSafeEqual` 校验；旧版本、篡改、过期、未知实例、错误成员和过期序号均被拒绝。
- 票据只保存在服务端实例记录，不进入浏览器/Godot 协议、快照或副本事件；重复验证只映射到原实例，不创建第二个副本。

## 2026-07-14 · 副本 Phase 3a：实例实体抽离

- 每个副本实例拥有独立的敌人、投射物和掉落集合；副本实体不再进入主 World 的共享集合。
- 普通 World tick 不再推进副本实体，副本地图快照仍从实例集合读取，实体清理、奖励和现有玩家可见协议保持兼容。
- 3a 暂不接入 child process tick；副本战斗输入与实体模拟留给后续 Phase 3b。

## 2026-07-14 · 副本 Phase 3b：Worker tick 与续接基础

- child worker 新增独立 `DungeonSimulation`，从副本 plan 和独立 `rngState` 加载实体，提供 `attach`、`detach`、`input` 和显式 `tick`。
- worker 消息改为串行处理；输入流和 tick 批次按玩家单调 `seq` 合并去重，返回快照、事件、状态版本和 RNG checkpoint 元数据。
- 新增真实 child process 测试覆盖 attach/detach、tick、重复输入和玩家状态续接；完整实体 checkpoint/restore 留给 Phase 4。

## 2026-07-14 · 副本 Phase 4：Checkpoint、Restore 与 Fencing

- 副本 worker checkpoint 现在保存完整 World 运行态、输入队列、剩余敌人和可序列化 RNG，可在同一或新 child process 恢复。
- 新 worker 续接后通过 `workerEpoch`、实例 ID、协议版本和 request ID 拒绝旧进程响应；真实 child 重放测试验证 snapshot、事件和 RNG 后续一致。

## 2026-07-13 · 功能对齐、副本与长期账号存储

- 浏览器与 Godot 共用新增的账号命令：七天一次性恢复码、会话令牌轮换和离线账号找回。服务端只持久化 SHA-256 摘要；官方客户端在凭据变更前先保存随机 `nextToken`，提交成功但响应丢失时可幂等重试；只有权威 `session` 成功落盘才清 pending，业务错误不会误删可能已提交的新 bearer，本地语法损坏的 pending 会先修复。`session` 回传权威职业以校正找回界面。安全命令全局串行执行，先完成持久化再回传 secret，失败则回滚内存凭据。浏览器修复离线找回误发普通登录和恢复码输入框 Enter 路径；Godot 对令牌保存错误给出可见提示、例行断线续接不再弹开账号面板，并在桌面 Unix 固定 `session.cfg` 为 0600。

- 新增纯函数生成的确定性组队副本「深红中继密库」：队首在所有成员在线、已认证、同图且存活时开启，按平均等级生成固定五守卫一首领阵列，成员奖励只结算一次。实例默认最多 32 个、15 分钟超时；超时、主动离开或城镇重生都会回收相应成员/怪物/投射物/掉落。

- 非主动断线新增默认 15 秒保席：暂停并隐藏玩家，但保留原 id、队伍与副本席位；同名有效 bearer 在窗口内接管原对象，错误 bearer 无法抢占。保席对象不能开启新副本，也不获取普通地图的组队 XP/任务进度。超时、显式离开或停服才执行完整清理；`WS_RECONNECT_GRACE_MS=0` 可禁用。

## 2026-07-13 · 存档加固：物品 ID、跨图商店、schema 版本与文件权限

- 修复重启后物品 ID 冲突：`item-N` 序号进程重启归零，而存档角色的背包/装备仍持有旧 ID，新掉落会铸出重复 ID（按 ID 出售/装备/使用会命中错误物品）。现在世界启动时扫描账号存档，序号从已用最大值之后继续；数值空间到安全边界时改用 UUID，不生成 schema 无法再次加载的边界 ID。
- 修复跨图买商店货：`buy` 只校验了 x/y 距离，不校验地图——站在其他地图上与城镇商人坐标重合处即可远程购买。现在商人携带所属地图，买家必须站在同一张图上（否则同样返回 `TOO_FAR`），协议不变。
- 存档文件引入 schema 版本信封：`{ "schema": 1, "savedAt": …, "accounts": { … } }`。旧的无版本平铺文件加载时自动迁移、下次保存原地升级；遇到比服务器更新的 schema 直接拒绝启动（不隔离、不改写），防止降级部署静默重写新格式；JSON 合法但整体结构错误的文件按损坏隔离并回退 `.bak`。账号记录逐条验证，单条损坏只从活动存档剔除并保留到 owner-only 的 `.invalid-records.json`，不会让同文件内其他角色丢档或拖垮快照广播。
- 存档权限收紧：`accounts.json`、`.tmp`、`.bak`、无效记录恢复文件与轮转备份统一 0600；默认/新建数据目录、备份目录和 systemd StateDirectory 统一 0700。显式 `PERSIST_PATH` 的已有父目录默认保留原权限，只有专用目录设置 `PERSIST_MANAGE_DIRECTORY=1` 后才强制为 0700。systemd 单元已开启该选项，并增加 `StateDirectoryMode=0700` 与 `UMask=0077`。

## 2026-07-11 · 点击操作、转生、安全区与红月主题

**操作方式（经典 aRPG 手感）**

- 左键点击地面移动，可按住拖动持续修正路线；服务器权威直线寻路，到达自动停止。
- 左键点击敌人锁定目标：自动走进普攻射程并持续攻击直到目标倒下，击杀后自动解除锁定。
- 右键或空格朝准星方向手动攻击；WASD/方向键手动移动会取消所有点击指令。
- 指令语义：`input` 消息中 `moveTo` / `target` 字段缺省表示保持现有指令，显式 `null` 表示取消。

**转生系统**

- 10 级解锁 `rebirth` 命令：等级归一、经验清零，换取永久 +6 属性点、每次转生 +12% 生命上限和 +15% 伤害。
- 等级旁显示 ★ 转生数，HUD 达标后出现「转生」按钮。

**城镇安全区**

- 地图中央半径 150 的安全区：怪物不索敌、不进入、无法造成伤害，区内恢复速度 4 倍。
- 地砖用暖色区分，边界为金色脉动圆环；`welcome` 与快照携带 `safeZone` 配置。

**世界与视觉**

- 野外怪物分为三种原创种类并按等级随机：裂隙体 Lv1、暮牙兽 Lv2、烬翼 Lv3。
